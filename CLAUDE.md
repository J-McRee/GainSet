# Gainset — Claude Code Master Instructions ## Project Overview Gainset is a self-hosted personal life management app built with: - Frontend: React + Vite + Tailwind CSS (in /client) - Backend: Node.js + Express (in /server) - Database: SQLite via Drizzle ORM - Deployment: Docker + Docker Compose ## Your Role You are building Gainset incrementally, one module at a time, in focused sessions. Each session should complete ONE scoped task and leave the codebase in a working state. Never leave broken imports, incomplete functions, or TODO placeholders without flagging them. ## Core Rules 1. Always read this file at the start of every session before writing any code. 2. Check /docs/phases/ for the current phase spec before starting work. 3. Run the app after each significant change to confirm it still starts. 4. Keep components small — if a file exceeds 200 lines, split it. 5. Write clear comments on any non-obvious logic. 6. Commit-ready means: no console errors, no broken routes, no placeholder UI. ## Database Rules - All schema changes go through Drizzle migrations — never edit the DB directly. - Run `npm run db:migrate` after any schema change. - Seed data goes in /server/src/db/seed.js. ## API Conventions - All endpoints prefixed with /api/v1/ - All responses: { success: boolean, data: any, error?: string } - Auth middleware applied to all routes except /api/v1/auth/\* ## Session Workflow 1. Read CLAUDE.md (this file) 2. Read the current phase doc in /docs/phases/ 3. Run `git status` to see current state 4. Complete the scoped task 5. Test that the app runs 6. Summarize what was built and what comes next ## Current Phase See /docs/phases/ folder for active phase. Start with phase-01-foundation.md.
